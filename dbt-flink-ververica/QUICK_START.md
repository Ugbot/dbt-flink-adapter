# dbt-flink-ververica Quick Start

**Deploy dbt-flink models to Ververica Cloud in 3 steps.**

---

## Installation

```bash
cd dbt-flink-ververica
pip install -e .
```

Or from the parent directory:
```bash
pip install -e dbt-flink-ververica/
```

---

## Quick Start (3 Steps)

### 1. Login

Save your Ververica Cloud credentials securely:

```bash
dbt-flink-ververica auth login --email your@email.com
```

Credentials are stored in your system keyring (macOS Keychain, Windows Credential Manager, Linux Secret Service).

### 2. Compile

Compile your dbt-flink models and transform them for Ververica:

```bash
dbt-flink-ververica compile \
  --project-dir ./my-dbt-project \
  --target prod
```

This will:
- Run `dbt compile`
- Parse dbt-flink query hints
- Generate SET statements
- Extract DROP statements
- Output clean SQL to `target/ververica/`

### 3. Deploy

Deploy to Ververica Cloud:

```bash
dbt-flink-ververica deploy \
  --name my-flink-job \
  --sql-file target/ververica/my_model.sql \
  --workspace-id YOUR_WORKSPACE_ID \
  --email your@email.com
```

Or use the **workflow** command to do it all in one step:

```bash
dbt-flink-ververica workflow \
  --name my-flink-job \
  --project-dir ./my-dbt-project \
  --workspace-id YOUR_WORKSPACE_ID \
  --email your@email.com
```

## What It Does

### SQL Transformation

**Before** (dbt-flink with hints):
```sql
/** mode('streaming') */
/** drop_statement('DROP TABLE IF EXISTS events') */

CREATE TABLE events (id BIGINT, name STRING)
WITH ('connector' = 'kafka', ...);
```

**After** (Ververica-ready):
```sql
-- SQL generated by dbt-flink-ververica

-- Configuration
SET 'execution.runtime-mode' = 'streaming';

-- Drop existing objects
DROP TABLE IF EXISTS events;

-- Main SQL
CREATE TABLE events (id BIGINT, name STRING)
WITH ('connector' = 'kafka', ...);
```

## All Commands

### Authentication

```bash
# Login and save credentials
dbt-flink-ververica auth login --email your@email.com

# Check credential status
dbt-flink-ververica auth status --email your@email.com

# Logout and delete credentials
dbt-flink-ververica auth logout --email your@email.com
```

### Configuration

```bash
# Create config file
dbt-flink-ververica config init --output dbt-flink-ververica.toml

# Validate config file
dbt-flink-ververica config validate dbt-flink-ververica.toml
```

### Compilation

```bash
# Compile all models
dbt-flink-ververica compile --project-dir ./my-project

# Compile specific models
dbt-flink-ververica compile --models model1,model2

# Custom output directory
dbt-flink-ververica compile --output-dir ./processed-sql

# Use different target
dbt-flink-ververica compile --target prod
```

### Deployment

```bash
# Deploy single SQL file
dbt-flink-ververica deploy \
  --name my-job \
  --sql-file ./my-model.sql \
  --workspace-id abc-123 \
  --email your@email.com

# Deploy to specific namespace
dbt-flink-ververica deploy \
  --name my-job \
  --sql-file ./my-model.sql \
  --workspace-id abc-123 \
  --namespace production \
  --email your@email.com

# Set parallelism
dbt-flink-ververica deploy \
  --name my-job \
  --sql-file ./my-model.sql \
  --workspace-id abc-123 \
  --parallelism 4 \
  --email your@email.com
```

### Workflow (Compile + Deploy)

```bash
# Full workflow
dbt-flink-ververica workflow \
  --name my-pipeline \
  --project-dir ./my-dbt-project \
  --workspace-id abc-123 \
  --email your@email.com

# Different environment
dbt-flink-ververica workflow \
  --name my-pipeline-prod \
  --project-dir ./my-dbt-project \
  --workspace-id prod-workspace \
  --namespace production \
  --target prod \
  --email your@email.com
```

## Common Workflows

### Development Loop

```bash
# 1. Edit dbt models
vim models/my_model.sql

# 2. Test locally
dbt run --select my_model

# 3. Deploy to dev environment
dbt-flink-ververica workflow \
  --name my-model-dev \
  --workspace-id dev-workspace \
```

```bash

  --name pipeline-prod \
  --workspace-id prod-ws \
```

## Configuration File

Create `dbt-flink-ververica.toml` to avoid repeating flags:

```toml
[ververica]
gateway_url = "https://app.ververica.cloud"
workspace_id = "your-workspace-id"
namespace = "default"

[dbt]
project_dir = "."
target = "prod"

[deployment]
deployment_name = "my-pipeline"
parallelism = 2

[deployment.flink_config]
"execution.checkpointing.interval" = "60s"
"state.backend" = "rocksdb"

[deployment.tags]
environment = "production"
team = "data-platform"
```

Then use it:
```bash
dbt-flink-ververica workflow \
  --config dbt-flink-ververica.toml \
  --email your@email.com
```

## Troubleshooting

### "No credentials found"

```bash
# Run auth login first
dbt-flink-ververica auth login --email your@email.com
```

### "dbt compile failed"

- Ensure dbt is installed: `pip install dbt-flink-adapter`
- Check profiles.yml is configured
- Try running `dbt compile` manually first

### "No compiled models found"

- Run `dbt compile` first
- Check that `target/run/` directory exists
- Verify models are being compiled (check `target/run/**/*.sql`)

### "Token expired"

Tokens auto-refresh, but if you see this:
```bash
# Re-authenticate
dbt-flink-ververica auth logout --email your@email.com
dbt-flink-ververica auth login --email your@email.com
```
